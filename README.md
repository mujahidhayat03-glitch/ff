## 📱 App Overview

**FF Pro Arena PK** is a full-stack, production-ready Free Fire tournament management app built with Flutter and Firebase. It supports real-time tournaments, in-app wallet (Easypaisa/JazzCash), live chat with admin moderation, VIP animations, and a full admin command center.

---

## 🚀 Feature Summary

| Feature                  | Details                                                                 |
|--------------------------|-------------------------------------------------------------------------|
| 🔐 Authentication         | Mobile + Password login, Firebase Auth with Firestore profiles          |
| 👤 Roles                  | User / VIP / Admin with distinct privileges                             |
| 🏆 Tournaments            | Create, join, manage — Bermuda, Purgatory, Kalahari, Alpine, Neextarra  |
| 💬 Global Chat            | Real-time text + image, admin moderation, typing indicators, pin/delete |
| 💰 Wallet                 | Easypaisa/JazzCash deposit with screenshot proof, Admin approval        |
| 🎖️ VIP Animations         | Golden Commander (admin) and Silver Crown (VIP) login triggers           |
| 📣 Broadcast              | Admin push notification + system message to all users                   |
| 🏠 Room ID Sharing        | Admin shares Room ID/Password 15 min before match — visible to joiners  |
| 🛡️ Security               | Firestore + Storage rules, role-based access                            |
| 🔔 Notifications          | Firebase Cloud Functions for deposit, room ID, and broadcast alerts     |

## ⚙️ Setup Guide

### Step 1 — Create Firebase Project

1. Go to [console.firebase.google.com](https://console.firebase.google.com)
2. Create project: **FF Pro Arena PK**
3. Enable these services:
   - ✅ **Authentication** → Email/Password (used with mobile-formatted emails)
   - ✅ **Cloud Firestore** → Start in production mode
   - ✅ **Firebase Storage**
   - ✅ **Cloud Messaging (FCM)**
   - ✅ **Cloud Functions** (requires Blaze plan)

---

### Step 2 — Flutter Setup

```bash
# Install Flutter & Dart (if not installed)
# https://docs.flutter.dev/get-started/install

# Clone / create the project
cd ff_pro_arena_pk

# Install FlutterFire CLI
dart pub global activate flutterfire_cli

# Connect to Firebase (auto-generates firebase_options.dart)
flutterfire configure

# Install dependencies
flutter pub get

# Run the app
flutter run
```

---

### Step 3 — Deploy Security Rules

```bash
# Install Firebase CLI
npm install -g firebase-tools
firebase login

# Deploy Firestore rules
firebase deploy --only firestore:rules

# Deploy Storage rules
firebase deploy --only storage

# Deploy Cloud Functions
cd functions
npm install
cd ..
firebase deploy --only functions
```

---

### Step 4 — Create Admin Account

1. Register a new account in the app
2. Go to Firebase Console → Firestore → `users` collection
3. Find your user document
4. Set `role` field to `"admin"`

The account now has full admin access including the golden command center.

---

### Step 5 — Android Configuration

Add to `android/app/src/main/AndroidManifest.xml`:
```xml
<!-- Permissions -->
<uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>
<uses-permission android:name="android.permission.CAMERA"/>
<uses-permission android:name="android.permission.RECORD_AUDIO"/>
<uses-permission android:name="android.permission.POST_NOTIFICATIONS"/>
```

## 🔒 Security Architecture

- **Firebase Auth** handles all authentication (no custom auth server needed)
- **Firestore Rules** enforce role-based access at the database level
- **Admin role** is set server-side (in Firestore), not client-side
- **Mobile numbers** are stored but only readable by Admin (enforced in rules + UI)
- **Wallet transactions** use Firestore atomic transactions to prevent race conditions
